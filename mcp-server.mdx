The Galaxy MCP server exposes workflow operations as tools for AI assistants. It is **hosted** — your client connects to a URL with your API key as a Bearer token. No clone, no script path, no local process, no database. Tools call the same public REST API (`https://galaxy-temp-frontend.vercel.app/api/v1`) you use directly, so auth, credits, and responses are identical.

## Prerequisites

- A `gx_` API key — create one in the **Settings** modal in the left sidebar of the canvas.
- An MCP client that supports remote (HTTP) servers — e.g. Cursor or Claude Desktop.

No local clone. No script path. No `DATABASE_URL`. No Trigger.dev credentials.

---

## 1. Get an API key

Sign into the canvas at `https://galaxy-temp-frontend.vercel.app`, open **Settings** from the left sidebar, and create a new API key under the **API Keys** tab. Copy it immediately — it is shown only once.

---

## 2. Connect your client

The server lives at:

```
https://galaxy-temp-frontend.vercel.app/api/mcp
```

Pass your API key as a Bearer token in the `headers` block — that's the only configuration.

<Tabs>
  <Tab title="Cursor">
    Edit `~/.cursor/mcp.json` (global) or `.cursor/mcp.json` (per-project):

    ```json
    {
      "mcpServers": {
        "galaxy": {
          "url": "https://galaxy-temp-frontend.vercel.app/api/mcp",
          "headers": {
            "Authorization": "Bearer gx_your_key_here"
          }
        }
      }
    }
    ```

    Restart Cursor after saving. The Galaxy tools appear in the MCP panel.
  </Tab>
  <Tab title="Claude Desktop">
    Edit your Claude Desktop config:

    - **macOS:** `~/Library/Application Support/Claude/claude_desktop_config.json`
    - **Windows:** `%APPDATA%\Claude\claude_desktop_config.json`

    ```json
    {
      "mcpServers": {
        "galaxy": {
          "url": "https://galaxy-temp-frontend.vercel.app/api/mcp",
          "headers": {
            "Authorization": "Bearer gx_your_key_here"
          }
        }
      }
    }
    ```

    Restart Claude Desktop after saving.
  </Tab>
</Tabs>

## 3. Verify it works

Ask your AI assistant: *"List my Galaxy workflows."*

Or test the endpoint directly with `curl` (a `tools/list` JSON-RPC call):

```bash
curl -s https://galaxy-temp-frontend.vercel.app/api/mcp \
  -H "Authorization: Bearer gx_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"tools/list"}'
```

## Available tools

| Tool | What it does | REST equivalent |
| --- | --- | --- |
| `list_workflows` | Metadata for all workflows owned by the key's user | `GET /api/v1/workflows` |
| `get_workflow` | Full `nodes` and `edges` JSON for one workflow | `GET /api/v1/workflows/{id}` |
| `create_workflow` | New workflow with default scaffold | `POST /api/v1/workflows` |
| `start_run` | Full-scope run — returns a `runId` to poll | `POST /api/v1/runs` |
| `get_run_status` | Run record plus per-node results | `GET /api/v1/runs/{id}` |
| `get_balance` | Microcredit balance | `GET /api/v1/credits/balance` |

### start_run

| Argument | Required | Description |
| --- | --- | --- |
| `workflowId` | Yes | ID of the workflow to run |
| `inputValues` | No | Flat object mapping Request-Inputs field IDs to values, e.g. `{ "field_text_default": "Hello" }` |

### create_workflow

Creates the same default scaffold as `POST /api/v1/workflows`:

- One `requestInputs` node with a `field_text_default` text field
- One `response` node with empty results
- No edges

After creating via MCP, open the canvas to build the full graph visually.

## Typical conversation flow

```
list_workflows()
  → sees "Image pipeline" (id: clxwf...)

start_run(workflowId: "clxwf...", inputValues: { field_text_default: "A red Ferrari" })
  → returns runId: "clxrun..."

get_run_status(runId: "clxrun...")
  → polls until status: "success"
  → returns nodeRuns with outputs
```

## Pointing at a local backend

Running Galaxy locally? Point the `url` at your local server instead:

```json
{
  "mcpServers": {
    "galaxy": {
      "url": "http://localhost:3000/api/mcp",
      "headers": { "Authorization": "Bearer gx_mock_your_key" }
    }
  }
}
```

## Troubleshooting

| Symptom | Fix |
| --- | --- |
| `Missing API key` | Add the `Authorization: Bearer gx_...` entry to the `headers` block of your server config |
| `Invalid API key` | Key may be wrong or revoked — create a new one in the canvas Settings |
| Run stays `running` indefinitely | Check the Trigger.dev dashboard; confirm the worker is connected |
| Insufficient credits error | Call `get_balance` before running expensive graphs |
| Tools not visible in Cursor | Restart Cursor after editing `mcp.json`; confirm the `url` is reachable |
