The Galaxy MCP server exposes workflow operations as tools for AI assistants. It is **hosted** — your client connects to a URL with your API key as a Bearer token. No clone, no script path, no local process, no database. Tools call the same public REST API (`https://galaxy-temp-frontend.vercel.app/api/v1`) you use directly, so auth, credits, and responses are identical.

## Prerequisites

- A `gx_` API key — create one in the **Settings** modal in the left sidebar of the canvas.
- An MCP client that supports remote (HTTP) servers — e.g. Cursor or Claude Desktop.

No local clone. No script path. No `DATABASE_URL`. No Trigger.dev credentials.

---

## 1. Get an API key

Sign into the canvas at `https://galaxy-temp-frontend.vercel.app`, open **Settings** from the left sidebar, and create a new API key under the **API Keys** tab. Copy it immediately — it is shown only once.

---

## 2. Connect your client

The server lives at:

```
https://galaxy-temp-frontend.vercel.app/api/mcp
```

Pass your API key as a Bearer token in the `headers` block — that's the only configuration.

<Tabs>
  <Tab title="Cursor">
    Edit `~/.cursor/mcp.json` (global) or `.cursor/mcp.json` (per-project):

    ```json
    {
      "mcpServers": {
        "galaxy": {
          "url": "https://galaxy-temp-frontend.vercel.app/api/mcp",
          "headers": {
            "Authorization": "Bearer gx_your_key_here"
          }
        }
      }
    }
    ```

    Restart Cursor after saving. The Galaxy tools appear in the MCP panel.
  </Tab>
  <Tab title="Claude Desktop">
    Edit your Claude Desktop config:

    - **macOS:** `~/Library/Application Support/Claude/claude_desktop_config.json`
    - **Windows:** `%APPDATA%\Claude\claude_desktop_config.json`

    ```json

            "Authorization": "Bearer gx_your_key_here"

    ```

    Restart Claude Desktop after saving.
  </Tab>

## 3. Verify it works

In Cursor, confirm **galaxy** shows six tools in **Settings → MCP** (green connected state).

Then ask in chat:

- *"Use Galaxy to list my workflows."*
- *"What's my Galaxy credit balance?"*

Or test with `curl`:

```bash
curl -s https://galaxy-temp-frontend.vercel.app/api/mcp \
  -H "Authorization: Bearer gx_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"tools/list"}'
```

## Available tools

Six tools are exposed. Each maps to the public REST API on the same host (`/api/v1/...`).

| Tool | Description | REST equivalent |
| --- | --- | --- |
| `list_workflows` | List workflow metadata (id, name, status) for your account | `GET /api/v1/workflows` |
| `get_workflow` | Full graph JSON (`nodes`, `edges`) for one workflow | `GET /api/v1/workflows/{id}` |
| `create_workflow` | Create a new canvas with default Request-Inputs + Response nodes | `POST /api/v1/workflows` |
| `start_run` | Start a full-scope run; returns a `runId` to poll | `POST /api/v1/runs` |
| `get_run_status` | Run status, credits, and per-node outputs/errors | `GET /api/v1/runs/{id}` |
| `get_balance` | Current microcredit balance | `GET /api/v1/credits/balance` |

### `list_workflows`

No arguments. Returns an array of workflow summaries.

### `get_workflow`

| Argument | Required | Description |
| --- | --- | --- |
| `workflowId` | Yes | Workflow ID from `list_workflows` |

### `create_workflow`

| Argument | Required | Description |
| --- | --- | --- |
| `name` | Yes | Display name for the new workflow |
| `description` | No | Optional description |

Creates the same default scaffold as the REST API:

- `requestInputs` node with one `field_text_default` text field
- `response` node with empty results
- No edges — connect and build the graph in the canvas after creation

### `start_run`

| Argument | Required | Description |
| --- | --- | --- |
| `workflowId` | Yes | Workflow to execute |
| `inputValues` | No | Request-Inputs field values, e.g. `{ "field_text_default": "Hello world" }` |

Runs the full workflow (same as Playground **Run**). Deducts credits and executes on Trigger.dev in the background.

### `get_run_status`

| Argument | Required | Description |
| --- | --- | --- |
| `runId` | Yes | Run ID returned by `start_run` |

Poll until `status` is `success` or `failed`. Response includes `nodeRuns` with per-node `output`, `error`, and `creditCost`.

### `get_balance`

No arguments. Returns `microcredits` and a human-readable `credits` string (millions).

## Typical conversation flow

```
list_workflows()
  → sees "Image pipeline" (id: clxwf...)

start_run(workflowId: "clxwf...", inputValues: { field_text_default: "A red Ferrari" })
  → returns runId: "clxrun..."

get_run_status(runId: "clxrun...")
  → polls until status: "success"
  → returns nodeRuns with outputs
```

## Local development

With the frontend dev server (`http://localhost:3001`), use the same path — it proxies `/api/mcp` to your local backend:

```json
{
  "mcpServers": {
    "galaxy": {
      "url": "http://localhost:3001/api/mcp",
      "headers": { "Authorization": "Bearer gx_mock_your_key" }
    }
  }
}
```

## Troubleshooting

| Symptom | Fix |
| --- | --- |
| `Missing API key` | Add `"Authorization": "Bearer gx_..."` under `headers` in your MCP config |
| `Invalid API key` | Create a new key in **Settings → API Keys** |
| Run stays `running` | Poll `get_run_status` again; long graphs can take minutes |
| Insufficient credits | Call `get_balance` first |
| Tools not visible | Restart Cursor after saving `mcp.json` |
