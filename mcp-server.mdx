The Galaxy MCP server is a StdIO adapter that exposes workflow operations as tools for AI assistants. It connects to the same PostgreSQL database and Trigger.dev orchestrator as the REST API — there is no separate execution path.

The server lives at `galaxy-temp-backend/scripts/mcp-server.ts`.

---

## What you can do

Once connected, your AI assistant can manage workflows and runs directly from a conversation.

## Prerequisites

- `galaxy-temp-backend` cloned and dependencies installed (`pnpm install`)
- A `.env.local` with `DATABASE_URL` and Trigger.dev credentials
- A valid API key (create one in the canvas dashboard under **API and Outbound Webhooks**)

---

## 1. Get an API key

Follow the same steps as for the REST API — see [Authentication](/authentication#creating-a-key). Use either a `gx_` or `gx_mock_` key depending on your environment.

---

## 2. Set environment variables

```env
GALAXY_API_KEY=gx_your_key_here
DATABASE_URL=postgresql://user:password@host:5432/dbname
TRIGGER_SECRET_KEY=...
TRIGGER_API_URL=...
```

`GALAXY_API_KEY` is resolved to a Clerk user ID using the same verification logic as the REST API.

---

## 3. Connect your client

<Tabs>
  <Tab title="Cursor">
    Edit `~/.cursor/mcp.json` (global) or `.cursor/mcp.json` (per-project):

    ```json
    {
      "mcpServers": {
        "galaxy-workflows": {
          "command": "pnpm",
          "args": ["mcp:start"],
          "cwd": "C:\\path\\to\\galaxy-temp-backend",
          "env": {
            "GALAXY_API_KEY": "gx_your_key_here"
          }
        }
      }
    }
    ```

    If `pnpm` is not on your system PATH, use `tsx` directly:

    ```json
    {
      "mcpServers": {
        "galaxy-workflows": {
          "command": "npx",
          "args": ["tsx", "scripts/mcp-server.ts"],
          "cwd": "C:\\path\\to\\galaxy-temp-backend",
          "env": {
            "GALAXY_API_KEY": "gx_your_key_here"
          }
        }
      }
    }
    ```

    Restart Cursor after saving. The Galaxy tools appear in the MCP panel.
  </Tab>

## Available tools

| Tool | What it does | REST equivalent |
| --- | --- | --- |
| `list_workflows` | Metadata for all workflows owned by the key's user | `GET /api/v1/workflows` |
| `get_workflow` | Full `nodes` and `edges` JSON for one workflow | `GET /api/v1/workflows/{id}` |
| `create_workflow` | New workflow with default scaffold | `POST /api/v1/workflows` |
| `start_run` | Full-scope run with credit hold and orchestrator | `POST /api/v1/runs` (scope: full) |
| `get_run_status` | Run record plus per-node results | `GET /api/v1/runs/{id}` |
| `get_balance` | Microcredit balance | `GET /api/v1/credits/balance` |

### start_run

| Argument | Required | Description |
| --- | --- | --- |
| `workflowId` | Yes | ID of the workflow to run |
| `inputValues` | No | Object mapping Request-Inputs field IDs to values |

The MCP server always starts **full-scope** runs. For partial or single-node reruns, use the REST API directly — see [Execution scopes](/execution-scopes).

### create_workflow

Creates the same default scaffold as `POST /api/v1/workflows`:

- One `requestInputs` node with a `field_text_default` text field
- One `response` node with empty results
- No edges

After creating via MCP, open the canvas to build the full graph visually.

---

## Typical conversation flow

```
list_workflows()
  → sees "Image pipeline" (id: clxwf...)

start_run(workflowId: "clxwf...", inputValues: { field_text_default: "A red Ferrari" })
  → returns runId: "clxrun..."

get_run_status(runId: "clxrun...")
  → polls until status: "success"
  → returns nodeRuns with outputs
```

## Troubleshooting

| Symptom | Fix |
| --- | --- |
| `GALAXY_API_KEY environment variable is not configured` | Add `GALAXY_API_KEY` to the `env` block in your MCP config |
| `Invalid GALAXY_API_KEY` | Key may be wrong or revoked — create a new one in the dashboard |
| Run stays `running` indefinitely | Check the Trigger.dev dashboard to see if the orchestrator task is stalled. Confirm the worker is connected |
| Insufficient credits error | Call `get_balance` before running expensive graphs |
| Tools not visible in Cursor | Restart Cursor after editing `mcp.json`. Verify the `cwd` path is correct and the backend dependencies are installed |
