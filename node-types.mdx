Node definitions live in `@galaxy/shared` (the `shared/` package inside `galaxy-temp-backend`). Each definition includes Zod schemas for inputs and outputs, UI field metadata, typed edge handle declarations, and a base credit cost.

## Scaffold nodes

Two nodes exist in every workflow by default. They are resolved **inline** by the orchestrator â€” no Trigger.dev task is spawned.

<CardGroup cols={2}>
  <Card title="requestInputs" icon="circle-arrow-right">
    The entry point for every workflow. Fields declared here become the keys in `inputValues` when you call `POST /api/v1/runs`.

    A default scaffold includes one text field: `field_text_default`. You can add more fields of different types through the canvas.

    **Base cost: 0 microcredits**
  </Card>
  <Card title="response" icon="circle-arrow-left">
    Declares which upstream node outputs appear as the workflow's final result. The `results` list on this node references source handles from connected nodes.

    **Base cost: 0 microcredits**
  </Card>
</CardGroup>

Field types for `requestInputs` and the value format each expects in `inputValues`:

| Field type | Value format |
| --- | --- |
| `text_field`, `text` | String |
| `number_field` | Number |
| `boolean_field` | Boolean |
| `image_field` (multi) | Array of image URL strings |
| `video_field` | Single URL string |
| `audio_field` | Single URL string |
| `file_field` | Single URL string |

## Working with nodes via API

The fastest workflow for building complex graphs:

<Steps>
  <Step title="Design visually">
    Build the graph in the canvas at `https://galaxy-temp-frontend.vercel.app`. Use drag-and-drop to add nodes and connect typed handles.
  </Step>
  <Step title="Export the JSON">
    Call `GET /api/v1/workflows/{id}` to retrieve the full `nodes` and `edges` arrays.
  </Step>
  <Step title="Use as template">
    Use that JSON as a starting point when creating or updating workflows programmatically via `POST` and `PUT /api/v1/workflows`.
  </Step>
</Steps>

Node objects stored in the database match the React Flow node shape:

```json
{
  "id": "gpt-image-node-1",
  "type": "gptImage2",
  "position": { "x": 400, "y": 200 },
  "data": {
    "label": "GPT Image 2",
    "prompt": "A retro synthwave city at dusk"
  }
}
```

Edges reference `source` and `target` node IDs with typed handle IDs (e.g. `image`, `text`, `video`). The handle type on the source must be compatible with the handle type on the target.

**Validation when updating via API:** the `nodes` array must be non-empty and must include at least one `requestInputs` node and one `response` node.
