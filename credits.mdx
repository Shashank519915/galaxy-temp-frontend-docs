title: Credits
description: Microcredit units, the hold-and-reconcile lifecycle, base costs per node type, and the balance API.

Galaxy uses a microcredit ledger to track usage. Every run places a hold at start and reconciles actual costs when the run finishes.

---

## Units

All balances and costs are expressed in **microcredits**:

| Microcredits | Display value |
| --- | --- |
| 1,000,000 | 1.00 credit |
| 210,000 | 0.21 credits |
| 100,000,000 | 100.00 credits |

`GET /api/v1/credits/balance` returns `{ "balance": <integer> }` in microcredits.

---

## Initial grant

When a user's balance is accessed for the first time, the system automatically creates a ledger entry of type `initial_grant` worth **100,000,000 microcredits** (100.00 credits). This happens on the first API call or dashboard login.

---

## Run lifecycle

<Steps>
  <Step title="Estimate">
    Before a run starts, `estimateWorkflowCost` sums the `credits.base` value from the shared node definition for each node in the execution set. This is an upfront estimate, not the final bill.
  </Step>
  <Step title="Hold">
    When `POST /api/v1/runs` is accepted (HTTP 202), the estimated cost is immediately deducted from the balance as a ledger entry of type `hold`.

    If the balance is insufficient, the request returns **500**:
    ```
    Insufficient credits. Estimated cost: 2.10M, but your balance is 0.50M.
    ```
  </Step>
  <Step title="Reconcile">
    When the orchestrator finishes, actual per-node costs from `NodeRun.creditCost` are recorded. The difference between the hold and the actual total is returned to the balance. This happens automatically.
  </Step>
</Steps>

<Tip>
Check `GET /api/v1/credits/balance` before starting runs on large graphs to avoid the insufficient credits error.
</Tip>

## Base costs by node type

| Node type | Base cost | Display |
| --- | --- | --- |
| `gptImage2` | 210,000 µcredits | 0.21 credits |
| `cropImage` | 210,000 µcredits | 0.21 credits |
| `gemini` | 450,000 µcredits | 0.45 credits |
| `openRouter` | 450,000 µcredits | 0.45 credits (see note) |
| `klingV3` | 1,500,000 µcredits | 1.50 credits |
| `mergeVideo` | 300,000 µcredits | 0.30 credits |
| `mergeAV` | 200,000 µcredits | 0.20 credits |
| `extractAudio` | 150,000 µcredits | 0.15 credits |
| `requestInputs` | 0 | Free |
| `response` | 0 | Free |

<Note>
`openRouter` has a base cost in its node definition but is not currently included in the upfront `estimateWorkflowCost` map. Credit holds for workflows containing OpenRouter nodes may therefore be lower than the actual provider cost. Actual cost is reconciled from `NodeRun.creditCost` after the run.
</Note>

The actual `creditCost` on each `NodeRun` reflects what the provider charged for that specific call and may differ from the base estimate.

## Checking balance

<Tabs>
  <Tab title="REST API">
    ```bash
    curl "https://galaxy-temp-frontend.vercel.app/api/v1/credits/balance" \
      -H "Authorization: Bearer YOUR_API_KEY"
    ```

    ```json
    { "balance": 84070000 }
    ```
  </Tab>
  <Tab title="MCP">
    Ask your AI assistant to call `get_balance` — it calls the same underlying function as the REST route.
  </Tab>
</Tabs>
