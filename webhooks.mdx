---
title: Webhooks
description: Receive signed HTTP POST events when runs start, nodes complete, and runs finish or fail.
---

Galaxy sends outbound webhook events to a URL you configure on a workflow. Delivery is asynchronous — events are fired from a dedicated Trigger.dev task (`emit-webhook`) so they do not block the orchestrator or the API response.

---

## Quick start

<Steps>
  <Step title="Configure a webhook URL">
    Send a `POST` to the dashboard route with a Clerk session:

    ```bash
    POST /api/workflows/{id}/webhook
    Content-Type: application/json

    { "webhookUrl": "https://your-server.com/galaxy-hooks" }
    ```

    On first save, the server generates a `webhookSecret` of the form `whsec_<32 hex chars>`. Copy it from the response — you need it to verify signatures.
  </Step>
  <Step title="Galaxy sends events to your URL">
    ```json
    {
      "success": true,
      "type": "run.completed",
      "runId": "clxrun...",
      "workflowId": "clxwf...",
      "data": { "status": "success", "durationMs": 42000, "outputs": { ... } },
      "error": null
    }
    ```
  </Step>
  <Step title="Verify and handle on your server">
    <Tabs>
      <Tab title="Node.js">
        ```javascript
        const crypto = require("crypto");

        // Use raw body — see verification section below
        app.post("/galaxy-hooks", (req, res) => {
          res.status(200).send("OK"); // respond immediately

          const isValid = verifyGalaxyWebhook(
            req.rawBody,
            req.headers,
            process.env.WEBHOOK_SECRET
          );
          if (!isValid) return;

          const event = req.body;
          if (event.type === "run.completed") {
            console.log("Outputs:", event.data.outputs);
          }
        });
        ```
      </Tab>
      <Tab title="Python">
        ```python
        @app.post("/galaxy-hooks")
        async def galaxy_webhook(request: Request):
            body = await request.body()
            is_valid = verify_galaxy_webhook(
                body, dict(request.headers), os.environ["WEBHOOK_SECRET"]
            )
            if not is_valid:
                return Response(status_code=400)

            event = await request.json()
            if event["type"] == "run.completed":
                print("Outputs:", event["data"]["outputs"])

            return {"ok": True}
        ```
      </Tab>
    </Tabs>
  </Step>
</Steps>

<Tip>
Always respond `200 OK` immediately and process asynchronously. If your handler takes too long, the Trigger.dev task will retry and you may receive duplicate deliveries.
</Tip>

## Configuration

Webhooks are configured **per workflow** through the dashboard. There is no `/api/v1` endpoint for webhook configuration — it requires a Clerk session.

`POST /api/workflows/{id}/webhook`

| Rule | Detail |
| --- | --- |
| URL format | Must start with `http://` or `https://` |
| Clear webhook | Send an empty string for `webhookUrl` |
| Secret rotation | If you update the URL on an existing webhook, the existing `webhookSecret` is preserved |
| Secret format | `whsec_<32 hex chars>` — generated on first save |

**Common patterns:**

| Use case | Subscribe to |
| --- | --- |
| Just the final result | `run.completed`, `run.failed` |
| Track per-node progress | `node.completed`, `run.completed`, `run.failed` |
| Failures only | `run.failed` |

## Delivery

- **Method:** POST
- **Content-Type:** application/json
- **Retries:** 3 attempts, exponential backoff starting at 5 seconds, max 60 seconds

Headers on every request:

| Header | Example |
| --- | --- |
| `svix-id` | `msg_a1b2c3d4e5f6a7b8` |
| `svix-timestamp` | `1717200000` |
| `svix-signature` | `v1,base64encodedhmac` |

<Warning>
Make your handler **idempotent**. A retry caused by a slow ACK can produce duplicate events — key on `runId` + `type` to detect repeats.
</Warning>

## Signature verification

Every event is signed with HMAC-SHA256. The algorithm:

```
signatureContent = "{svix-id}.{svix-timestamp}.{raw request body}"
HMAC-SHA256(webhookSecret, signatureContent) → base64
```

<Warning>
Verify using the **raw request body bytes**, not a re-parsed JSON object. If your framework parses the body before your handler runs (Express, Fastify, etc.), use middleware that captures the raw buffer first (`express.raw({ type: 'application/json' })`). The signature will not match if the body has been re-serialized.
</Warning>

Reject events with a `svix-timestamp` more than **5 minutes** old to prevent replay attacks.

<Tabs>
  <Tab title="Node.js">
    ```javascript
    const crypto = require("crypto");

    function verifyGalaxyWebhook(rawBody, headers, webhookSecret) {
      const svixId = headers["svix-id"];
      const svixTimestamp = headers["svix-timestamp"];
      const svixSignature = headers["svix-signature"];

      const now = Math.floor(Date.now() / 1000);
      if (Math.abs(now - Number(svixTimestamp)) > 300) {
        throw new Error("Timestamp too old — possible replay attack");
      }

      const signatureContent = `${svixId}.${svixTimestamp}.${rawBody}`;
      const expected = crypto
        .createHmac("sha256", webhookSecret)
        .update(signatureContent)
        .digest("base64");

      const received = svixSignature.replace(/^v1,/, "");
      return crypto.timingSafeEqual(
        Buffer.from(received, "base64"),
        Buffer.from(expected, "base64")
      );
    }
    ```
  </Tab>
  <Tab title="Python">
    ```python
    import hashlib, hmac, time, base64

    def verify_galaxy_webhook(raw_body: bytes, headers: dict, webhook_secret: str) -> bool:
        svix_id = headers.get("svix-id", "")
        svix_timestamp = headers.get("svix-timestamp", "")
        svix_signature = headers.get("svix-signature", "")

        now = int(time.time())
        if abs(now - int(svix_timestamp)) > 300:
            raise ValueError("Timestamp too old")

        body_str = raw_body.decode("utf-8")
        content = f"{svix_id}.{svix_timestamp}.{body_str}"
        expected = base64.b64encode(
            hmac.new(webhook_secret.encode(), content.encode(), hashlib.sha256).digest()
        ).decode()

        received = svix_signature.replace("v1,", "")
        return hmac.compare_digest(received, expected)
    ```
  </Tab>
</Tabs>

## Event payloads

All events share the same envelope:

| Field | Type | Description |
| --- | --- | --- |
| `success` | Boolean | `true` for success events, `false` for failures |
| `type` | String | Event name |
| `runId` | String | Workflow run ID |
| `workflowId` | String | Parent workflow ID |
| `data` | Object | Event-specific payload |
| `error` | String or null | Error message on failures; null otherwise |
