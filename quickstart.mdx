title: Quickstart

---

The Galaxy REST API is at `https://galaxy-temp-frontend.vercel.app/api/v1`. All examples use that base URL. Replace `YOUR_API_KEY` with a real key from the dashboard.

---

## 1. Get an API key

<Steps>
  <Step title="Sign in">
    Open `https://galaxy-temp-frontend.vercel.app` and sign in.
  </Step>
  <Step title="Open API and Outbound Webhooks">
    Click **API and Outbound Webhooks** in the left sidebar.
  </Step>
  <Step title="Create a key">
    Click **Create key**, give it a label, and copy the full `gx_...` token immediately. It is only shown once.
  </Step>
</Steps>

<Warning>
API keys cannot be retrieved after creation. If you lose one, revoke it and create a new key. Never commit keys to git or paste them in screenshots.
</Warning>

---

## 2. Check your balance

A read-only call to verify the key works.

<Tabs>
  <Tab title="cURL">
    ```bash
    curl "https://galaxy-temp-frontend.vercel.app/api/v1/credits/balance" \
      -H "Authorization: Bearer YOUR_API_KEY"
    ```
  </Tab>
  <Tab title="JavaScript">
    ```javascript
    const res = await fetch(
      "https://galaxy-temp-frontend.vercel.app/api/v1/credits/balance",
      { headers: { Authorization: `Bearer ${process.env.GALAXY_API_KEY}` } }
    );
    const { balance } = await res.json();
    console.log(balance); // e.g. 100000000
    ```
  </Tab>
  <Tab title="Python">
    ```python
    import os, requests

    res = requests.get(
        "https://galaxy-temp-frontend.vercel.app/api/v1/credits/balance",
        headers={"Authorization": f"Bearer {os.environ['GALAXY_API_KEY']}"},
    )
    print(res.json()["balance"])  # e.g. 100000000
    ```
  </Tab>
</Tabs>

```json
{ "balance": 100000000 }
```

`balance` is in **microcredits** — `100000000` is 100.00 credits, the default grant for new users. A `401` here means the key is missing, malformed, or revoked. See [Authentication](/authentication).

## 3. Create a workflow

<Tabs>
  <Tab title="cURL">
    ```bash
    curl -X POST "https://galaxy-temp-frontend.vercel.app/api/v1/workflows" \
      -H "Authorization: Bearer YOUR_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{"name": "My first pipeline", "description": "Created via API"}'
    ```
  </Tab>
  <Tab title="JavaScript">
    ```javascript
    const res = await fetch(
      "https://galaxy-temp-frontend.vercel.app/api/v1/workflows",
      {
        method: "POST",
        headers: {
          Authorization: `Bearer ${process.env.GALAXY_API_KEY}`,
          "Content-Type": "application/json",
        },
        body: JSON.stringify({ name: "My first pipeline" }),
      }
    );
    const { data } = await res.json();
    console.log(data.id); // save this as WORKFLOW_ID
    ```
  </Tab>
  <Tab title="Python">
    ```python
    res = requests.post(
        "https://galaxy-temp-frontend.vercel.app/api/v1/workflows",
        headers={
            "Authorization": f"Bearer {os.environ['GALAXY_API_KEY']}",
            "Content-Type": "application/json",
        },
        json={"name": "My first pipeline"},
    )
    workflow_id = res.json()["data"]["id"]
    ```
  </Tab>
</Tabs>

The response includes a `data.id` — save it as `WORKFLOW_ID`. The created workflow contains a `requestInputs` node (with one text field, `field_text_default`) and a `response` node, connected by no edges yet.

## 4. Design the graph (optional)

The fastest way to build a real graph is visually. Open the canvas, drag in nodes, wire them up, then fetch the JSON:

```bash
curl "https://galaxy-temp-frontend.vercel.app/api/v1/workflows/WORKFLOW_ID" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

Copy the `nodes` and `edges` arrays and `PUT` them onto any workflow:

```bash
curl -X PUT "https://galaxy-temp-frontend.vercel.app/api/v1/workflows/WORKFLOW_ID" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"name": "Updated name", "nodes": [...], "edges": [...]}'
```

Validation rules: `name` is 1–120 characters. If `nodes` is sent, it must be non-empty and include both a `requestInputs` and a `response` node. See [Node types](/node-types) for valid `type` values.

## 5. Start a run

<Tabs>
  <Tab title="cURL">
    ```bash
    curl -X POST "https://galaxy-temp-frontend.vercel.app/api/v1/runs" \
      -H "Authorization: Bearer YOUR_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "workflowId": "WORKFLOW_ID",
        "scope": "full",
        "inputValues": {
          "field_text_default": "A retro synthwave racing car at sunset"
        }
      }'
    ```
  </Tab>
  <Tab title="JavaScript">
    ```javascript
    const run = await fetch(
      "https://galaxy-temp-frontend.vercel.app/api/v1/runs",
      {
        method: "POST",
        headers: {
          Authorization: `Bearer ${process.env.GALAXY_API_KEY}`,
          "Content-Type": "application/json",
        },
        body: JSON.stringify({
          workflowId: WORKFLOW_ID,
          scope: "full",
          inputValues: { field_text_default: "A retro synthwave racing car at sunset" },
        }),
      }
    ).then(r => r.json());

    console.log("runId:", run.data.runId);
    ```
  </Tab>
  <Tab title="Python">
    ```python
    run = requests.post(
        "https://galaxy-temp-frontend.vercel.app/api/v1/runs",
        headers={
            "Authorization": f"Bearer {os.environ['GALAXY_API_KEY']}",
            "Content-Type": "application/json",
        },
        json={
            "workflowId": workflow_id,
            "scope": "full",
            "inputValues": {"field_text_default": "A retro synthwave racing car at sunset"},
        },
    ).json()

    run_id = run["data"]["runId"]
    ```
  </Tab>
</Tabs>

A successful response is **HTTP 202**:

```json
{
  "data": {
    "runId": "clxrun...",
    "status": "running",
    "orchestratorRunId": "run_abc123"
  }
}
```

The API immediately places a credit hold, creates the run record, and hands the job to Trigger.dev. Your call returns right away — poll separately for results.

Only one run can be active per workflow at a time. A concurrent second request returns **409** with the existing `runId`.

## 6. Poll for results

<Tabs>
  <Tab title="Poll for status">
    Hit `GET /api/v1/runs/{runId}` until status becomes `success`, `failed`, or `partial`.

    ```bash
    curl "https://galaxy-temp-frontend.vercel.app/api/v1/runs/RUN_ID" \
      -H "Authorization: Bearer YOUR_API_KEY"
    ```

    The `data.nodeRuns` array has one entry per graph node with `status`, `output`, `error`, `creditCost`, and `providerUsed`.
  </Tab>
  <Tab title="Subscribe via webhook">
    Configure a `webhookUrl` on the workflow once (dashboard only) and receive push events instead:

    | Event | When |
    | --- | --- |
    | `run.started` | After 202 accepted |
    | `node.completed` | After each node |
    | `run.completed` | All nodes finished |
    | `run.failed` | Run ended with failures |

    See [Webhooks](/webhooks) for signature verification and payload shapes.
  </Tab>
</Tabs>
