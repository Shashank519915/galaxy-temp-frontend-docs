---
title: Errors
description: HTTP status codes and error bodies returned by the public API.
---

Public v1 handlers return JSON `{ "error": string }` unless noted. Validation failures add `issues` (Zod).

## Status code reference

| Code | When | Example `error` |
| --- | --- | --- |
| **400** | Invalid JSON / Zod validation | `Invalid input` + `issues[]` |
| **401** | Missing/invalid Bearer | `Missing Authorization header`, `Invalid or unauthorized API key`, `Invalid API key` |
| **404** | Wrong user or missing resource | `Workflow not found`, `Run not found` |
| **409** | Concurrent run | `A run is already in progress for this workflow` (+ `runId`) |
| **429** | Unkey rate limit | `Rate limit exceeded` |
| **500** | Server error or business rule | `Internal server error`, insufficient credits message |

## Validation (`400`)

```json
{
  "error": "Invalid input",
  "issues": [
    {
      "code": "custom",
      "message": "Graph must include requestInputs and response nodes",
      "path": ["nodes"]
    }
  ]
}
```

Common Zod messages from `lib/validation.ts`:

- `Refusing to persist an empty graph`
- `Graph must include requestInputs and response nodes`
- `name` length 1–120

## Authentication (`401`)

`lib/api-auth.ts` behavior:

1. Requires `Authorization: Bearer <token>`.
2. If Unkey env vars set and key is not `gx_mock_*`, calls `https://api.unkey.com/v2/keys.verifyKey`.
3. On Unkey failure or mock keys, falls back to `apiKey` table hash lookup.
4. Unkey valid but missing `identity.externalId` → `API key is valid but has no identity/owner configured`.

## Conflict (`409`)

```json
{
  "error": "A run is already in progress for this workflow",
  "runId": "clxrun…"
}
```

Wait for that run to finish or cancel via dashboard (`POST /api/workflows/{id}/cancel`).

## Insufficient credits (`500`)

Not a separate HTTP code today—the runs handler throws inside the transaction:

```text
Insufficient credits. Estimated cost: 2.10M, but your balance is 0.50M.
```

Check balance with `GET /api/v1/credits/balance` before large graphs.

## Rate limit headers

On successful v1 responses (when Unkey returns limit metadata):

| Header | Meaning |
| --- | --- |
| `X-RateLimit-Limit` | Max requests in window |
| `X-RateLimit-Remaining` | Remaining requests |
| `X-RateLimit-Reset` | Unix seconds when window resets |

Mock/local mode returns a static budget (60/min) from `api-auth.ts`.

## Webhook delivery failures

Outbound webhooks run in Trigger.dev task `emit-webhook` with **3 retries** and exponential backoff (5s–60s). Failures are logged in Trigger.dev—not returned to the API caller.
