---
title: Introduction
description: Galaxy is a visual DAG workflow builder for chaining AI and media nodes, with a public REST API, outbound webhooks, and an MCP server for AI assistants.
---

Galaxy lets you build multi-step AI pipelines on a drag-and-drop canvas and run them via the dashboard, API, or an AI assistant like Cursor.

A pipeline is a **directed acyclic graph (DAG)** of nodes connected by typed edges. Each node is a discrete operation: generate an image with GPT Image 2, produce a video with Kling v3, run a prompt through Gemini or OpenRouter, or process media with FFmpeg utilities (crop, merge, audio extraction). Nodes pass typed outputs — text, image URL, video URL, audio URL — downstream through typed handles. The orchestrator on Trigger.dev walks the graph in topological order, running heavy nodes as isolated tasks with automatic retries.

## What you can do

<CardGroup cols={2}>
  <Card title="REST API" icon="code" href="/quickstart">
    Versioned `/api/v1/*` routes authenticated with `gx_` Bearer keys. Create workflows, start runs, poll node-level results, and read credit balance. The API Reference tab has an interactive playground.
  </Card>
  <Card title="DAG Execution" icon="circle-play" href="/architecture">
    One Trigger.dev coordinator task handles every run regardless of whether it came from the dashboard, the API, or MCP. Nodes run in topological order with waitpoint-based concurrency.
  </Card>
  <Card title="Outbound Webhooks" icon="webhook" href="/webhooks">
    Receive signed HTTP POSTs when runs start, each node completes, or a run finishes or fails. Signature format matches Svix verification libraries.
  </Card>
  <Card title="MCP Server" icon="plug" href="/mcp-server">
    Wire Galaxy into Cursor or Claude Desktop over StdIO. List workflows, create them, start full runs, and poll status without leaving your editor.
  </Card>
</CardGroup>

## Recommended path

<Steps>
  <Step title="Get an API key">
    Sign into the canvas, open **API and Outbound Webhooks** in the sidebar, and create a `gx_...` key. Copy it immediately — it is shown only once.
  </Step>
  <Step title="Make your first call">
    Follow the [Quickstart](/quickstart) to call `GET /api/v1/credits/balance` and create your first workflow via API.
  </Step>
  <Step title="Design a graph">
    Build a more complex pipeline in the canvas, then export the node/edge JSON and `PUT` it back via API.
  </Step>
  <Step title="Add a webhook">
    Configure a webhook URL on the workflow to receive real-time progress events during runs instead of polling.
  </Step>
  <Step title="Connect MCP (optional)">
    Set up the [MCP server](/mcp-server) to control workflows from Cursor or Claude Desktop.
  </Step>
</Steps>
